INTRODUCTION

Looking for an api for application development? I always found dev-teams always get confused while choosing a technology stack for an application whether it is a web or native one. In this era of tech-innovation we are quite confused between what is best and what is required. It is still a major concern to select a api for the application development,so let us briefly explore the possibilities of using the functionalities of graphql in your application.

Why GraphQL

I am quite sure why is a better question than what.As REST and SOAP is already trending in market so why companies like github,twitter,facebook,spotify chosing this api as per requirements is a nice question to explore.So some of the good reasons are asper follows :-

  • A single endpoint with a "POST" request is enough.GraphQL doesn't look for different operations like create,insert,delete etc. ,all such things can be done with a single post request following some arichitecure that is defined by graphql community different for different environments.
  • With a single query you can search for multiple databases.
  • It has a lot of adventage on reducing network payload as on fetching you will only get required data which varies from REST where you will always get a full document instead of few attributes of it.

So lets explore GraphQL by understanding its architecture and workflow.We will discuss it considering a mongoose Schema with express.js

Prerequisites

You should have node environment[install node.js + express either from commandline or terminal] and some basic knowledge on MongoDB and Javascript.Run the following commands for the final environment setup.

  • npm install express express-graphql graphql

Building apis at backend

Let us go through the following steps to understand graphql as well as the architecture

STEP - I [The express graphql structure]


const express = require('express');
const bodyParser = require('body-parser');
const graphqlHttp = require('express-graphql').graphqlHTTP;
const { buildSchema } = require('graphql');
const mongoose = require('mongoose');
const app = express();
app.use(bodyParser.json());

const graphQlSchema = require('./apis/schema/schemas');
const graphQlResolvers = require('./apis/resolver/resolvers');

app.use(
  '/graphql',
  graphqlHttp({
    schema: graphQlSchema,
    rootValue: graphQlResolvers,
    graphiql: true
})
)
mongoose.connect("put the connection string of database here",{useNewUrlParser: true,useUnifiedTopology: true,})
.then(() => { app.listen(3000); })
.catch(err => { console.log(err);
});
